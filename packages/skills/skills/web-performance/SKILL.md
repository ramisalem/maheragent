# Web performance

1. Capture a **baseline** with the performance tool before making any changes, so improvements are measurable.
2. Summarize the key signals — Core Web Vitals (LCP, CLS, INP), long tasks, and heavy network or render work.
3. Propose concrete, specific optimizations tied to what the profile actually showed — not generic advice.
4. After a change, re-profile and compare against the baseline.
